# Описание проекта для курсовой работы

## Тема: Использование искусственного интеллекта в предварительной установке диагноза: гастрит

## Краткое описание

Разработана система предварительной диагностики гастрита на основе анализа симптомов пациента с использованием методов машинного обучения и нейронных сетей. Система позволяет оценить вероятность наличия гастрита по 10 ключевым симптомам и предоставляет рекомендации по дальнейшим действиям.

## Цели и задачи

### Цель работы
Разработать и протестировать модель машинного обучения для предварительной диагностики гастрита на основе симптомов пациента.

### Задачи исследования
1. Провести анализ симптомов гастрита и их значимости для диагностики
2. Исследовать существующие архитектуры нейронных сетей для задач классификации
3. Собрать и подготовить датасет симптомов пациентов
4. Разработать архитектуру нейронной сети для диагностики
5. Обучить и оценить качество модели
6. Создать пользовательский интерфейс для диагностики
7. Разработать систему визуализации результатов

## Техническая реализация

### Используемые технологии
- **Язык программирования:** Python 3.8+
- **Фреймворк машинного обучения:** TensorFlow/Keras
- **Библиотеки обработки данных:** NumPy, Pandas, Scikit-learn
- **Библиотеки визуализации:** Matplotlib, Seaborn

### Архитектура модели

Модель представляет собой полносвязную нейронную сеть следующей архитектуры:

```
Входной слой (10 нейронов)
    ↓
Полносвязный слой (64 нейрона) + ReLU + Dropout(0.3)
    ↓
Полносвязный слой (32 нейрона) + ReLU + Dropout(0.2)
    ↓
Полносвязный слой (16 нейронов) + ReLU + Dropout(0.1)
    ↓
Выходной слой (1 нейрон) + Sigmoid
```

### Анализируемые симптомы

Система анализирует следующие симптомы (оцениваются по шкале от 0 до 10):

1. **Боль в животе** - один из основных симптомов гастрита
2. **Изжога** - чувство жжения за грудиной
3. **Тошнота** - неприятное ощущение в области желудка
4. **Рвота** - рефлекторное извержение содержимого желудка
5. **Потеря аппетита** - снижение интереса к пище
6. **Вздутие живота** - ощущение переполненности желудка
7. **Отрыжка** - непроизвольное выделение газов из желудка
8. **Слабость** - общее недомогание
9. **Потеря веса** - снижение массы тела
10. **Дискомфорт после еды** - неприятные ощущения после приема пищи

## Структура проекта

### Основные модули

1. **dataset.py** - модуль для генерации и подготовки данных
   - Класс `GastritisDataset` для работы с датасетом
   - Генерация синтетических данных симптомов
   - Подготовка данных для обучения (нормализация, разделение на выборки)

2. **model.py** - модуль для создания и обучения модели
   - Класс `GastritisModel` для работы с нейронной сетью
   - Создание архитектуры модели
   - Обучение с использованием callbacks (Early Stopping, Model Checkpoint)
   - Оценка качества модели
   - Сохранение и загрузка модели

3. **train.py** - скрипт для обучения модели
   - Полный цикл обучения модели
   - Генерация визуализаций результатов
   - Сохранение метрик качества

4. **predict.py** - скрипт для предсказания диагноза
   - Класс `GastritisPredictor` для предсказаний
   - Интерактивный режим диагностики
   - Пакетное предсказание для нескольких пациентов

5. **visualization.py** - модуль для визуализации результатов
   - График истории обучения
   - Матрица ошибок
   - График важности признаков
   - Сравнение симптомов пациентов
   - Визуализация вероятности предсказания
   - График метрик качества

6. **main.py** - главный файл приложения
   - Интерактивное меню
   - Объединение всех функциональностей
   - Удобный пользовательский интерфейс

## Метрики качества модели

Модель оценивается по следующим метрикам:

- **Accuracy** (Точность) - доля правильных предсказаний
- **Precision** (Точность положительных предсказаний) - способность модели избегать ложноположительных результатов
- **Recall** (Полнота) - способность модели находить все положительные примеры
- **F1-Score** - гармоническое среднее Precision и Recall

## Ограничения и рекомендации

### Ограничения системы
1. Используются синтетические данные для обучения
2. Результаты являются предварительными
3. Не учитываются индивидуальные особенности пациента
4. Не заменяет полноценное медицинское обследование

### Рекомендации по использованию
1. Оценивайте симптомы объективно по шкале от 0 до 10
2. При вероятности > 80% рекомендуется консультация врача
3. При вероятности 60-80% также рекомендуется консультация
4. При любых симптомах рекомендуется обратиться к врачу-гастроэнтерологу
